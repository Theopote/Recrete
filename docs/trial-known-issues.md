# 试用阶段已知问题清单

> 第 5 个月试用前透明告知事务所；试用后根据反馈更新。  
> **P0** = 阻塞试用 · **P1** = 严重 · **P2** = 体验 · **P3** = 后续版本

## 基础设施

| 级别 | 问题 | 说明 / 规避 |
|------|------|-------------|
| P0 | Vercel + 本地存储 | 未配置 S3/R2 时上传文件不持久。生产试用必须 `STORAGE_PROVIDER=s3` |
| P1 | 数据库迁移 | 新模型（试用反馈、方案留言等）需对生产库执行 `npm run db:push` |
| P2 | 冷启动 | Neon 免费档首次请求可能慢 1–2 秒 |

## 账号与协作

| 级别 | 问题 | 说明 / 规避 |
|------|------|-------------|
| P2 | 自助邀请同事 | 无邮件邀请流；新用户注册进默认 org，需运营方改角色/组织 |
| P2 | 注册默认 viewer | 需将试用账号升为 architect 才能生成方案、上传文档 |
| P3 | 正式 RBAC / 成员管理 | 路线图阶段二之后 |

## AI 与文档

| 级别 | 问题 | 说明 / 规避 |
|------|------|-------------|
| P1 | 无 OpenAI Key | 自动 mock，无法评估真实 AI 质量 — 试用前务必配置 Key |
| P2 | AI 额度 | 超 `AI_DAILY_ORG_LIMIT` / 月限时友好报错，可临时调高 |
| P2 | 大 PDF / DWG | 分析耗时；多页 PDF 最多 5 页；复杂 DWG 可能失败 |
| P2 | 后台任务 | 无 Redis 时靠内部 API 单实例派发，多实例需 `REDIS_URL` + worker |
