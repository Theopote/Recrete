# 试用观察手册 · 运营方（你）用

> 第 5 个月：找 1 家熟悉、愿意包容 Bug 的事务所。你全程盯会话，记录卡点。**本月不加新功能**，除非某环节完全用不下去。

## 试用前（T-3 天）

### 部署检查

```bash
# 本地或 CI：核心逻辑
npm run smoke:core
npm test

# 指向生产环境变量后：
USE_DATABASE=true \
DATABASE_URL="..." \
NEXTAUTH_URL="https://your-app.vercel.app" \
NEXTAUTH_SECRET="..." \
JOB_RUNNER_SECRET="..." \
OPENAI_API_KEY="..." \
STORAGE_PROVIDER="s3" \
S3_BUCKET="..." \
npm run smoke:trial-prep
```

- [ ] `db:push` + `db:seed` 已在生产库执行过一次
- [ ] 演示账号可登录；已为试用事务所创建独立 org（或确认注册账号可升级角色）
- [ ] `OPENAI_API_KEY` 已配置（否则 AI 为 mock，无法评估真实价值）
- [ ] `STORAGE_PROVIDER=s3`（Vercel 必配）
- [ ] `NEXT_PUBLIC_TRIAL_FEEDBACK` 未设为 `false`

### 材料

- [ ] 发送 [trial-pilot-guide.md](./trial-pilot-guide.md) 给事务所负责人
- [ ] 约定 1–2 次 **同步观察会**（屏幕共享），其余时间异步
- [ ] 准备 1 个真实项目背景话术（若对方暂无现成案例）

每场会话记录（可复制到表格）：

| | | 上传文档 | | | |
| | | 资料分析 | | | |
| | | 生成方案 | | | |
| | | 方案评审留言 | | | |

### 重点盯这三类信号

1. **卡点**：停住超过 2 分钟、反复点同一按钮、放弃当前路径
2. **看不懂的文案**：英文术语、按钮含义、空状态提示
3. **AI 价值**：方案是否像「能拿去讨论的方向」还是「泛泛而谈」

### 你可以问的话

- 「这一步您预期会发生什么？」
- 「如果这是您真实项目，这份方案您会采纳哪一条？为什么？」
- 「有没有一句界面文字让您犹豫？」

### 不要做的事

- 不要替用户点完全程（除非阻塞性 Bug）
- 不要现场加功能说服对方「其实还能……」
- 不要同时拉多家事务所（第一轮 1 家即可）

## 反馈汇总

### 应用内

- 用户：右下角 **试用反馈**
- 你：`Settings` → **试用反馈汇总**（admin / PM）或下载 CSV：
  `/api/trial-feedback?format=csv`

### 会话后整理

按优先级归档：

| 优先级 | 定义 | 处理月份 |
|--------|------|----------|
| P0 | 阻塞：无法完成建项目或生成方案 | 立即 hotfix |
| P1 | 严重误导或数据错误 | 第 6 个月首批 |
| P2 | 体验摩擦、文案 | 第 6 个月 |
| P3 | 愿望清单 |  backlog |

## 试用结束（T+1 周）

- [ ] 导出全部试用反馈 CSV
- [ ] 写 1 页 **试用结论**（3 条做得好的、3 条必须改的）
- [ ] 更新 [trial-known-issues.md](./trial-known-issues.md)
- [ ] 决定是否关闭反馈按钮：`NEXT_PUBLIC_TRIAL_FEEDBACK=false`
