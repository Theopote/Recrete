# 试用事务所上手指南 · Recrete 砼憶

> 面向事务所真实试用。请把本文发给事务所负责人，并附上公网地址与账号。

## 这是什么

Recrete（砼憶）是面向**既有建筑更新**的 AI 设计助手 MVP。当前版本可完成：

1. 用一句话描述项目背景，自动建项目
2. 上传旧图纸 / 照片 / 文档，异步 AI 解读
3. 生成多档改造方案并对比（含方案依据链）
4. 在方案卡片下留言评审（可 @ 同事）
5. （可选）BIM 模型查看、文保专家诊断、报告导出

**这是试用版**：可能有 Bug、部分功能仍在完善。界面支持**中/英切换**（右上角）。遇到问题请点右下角 **「试用反馈」** 按钮，或直接联系我方对接人。

## 推荐试用路径（约 30–45 分钟）

### 1. 登录

- 打开我方提供的公网地址
- 使用分配的邮箱与密码登录
- 若忘记密码：登录页 → 忘记密码
- 可在右上角切换 **中文 / English**

### 2. 建项目（核心体验）

1. 点击 **新建项目** 或侧边栏进入建项目页
2. 用中文或英文写一段项目背景，例如：
   > 我有一栋 1980 年代建成的框架办公楼，位于西安，原政府办公，拟改为社区文化中心，预算有限，希望保留主体结构。
3. 等待 AI 流式生成完成
4. 应自动进入项目的 **策略实验室**，看到欢迎提示与方案生成入口

### 3. 上传资料（可选但推荐）

1. 进入项目 → **调研情报** 分区
2. 上传 PDF 图纸、现场照片或 DWG（单文件建议 &lt; 20MB）
3. 点击 **分析**，等待异步分析完成（大文件可能需要 1–3 分钟）
4. 查看 AI 提取的要点是否与您认知一致

### 4. 生成改造方案

1. 在 **策略实验室** 点击生成 / 刷新方案
2. 查看三档方案（轻量 / 中等 / 深度）及 **AI 推荐理由**
3. 展开对比维度：造价、工期、风险、空间策略等
4. 查看方案卡片上的 **方案依据链**（关联的诊断与证据）
5. 在方案卡片下 **留言**，试 @ 一位同事并关联风险标签

### 5. 提交反馈

右下角 **试用反馈** 按钮，请选择类型并说明：

- **卡在某一步**：具体页面与操作
- **文案看不懂**：请粘贴原文
- **AI 方案是否有用**：1–5 分 + 简短说明
- 若完全无法继续，勾选 **阻塞性问题**

## 账号与权限说明

- 新注册用户默认为 **viewer**，可浏览与留言
- 生成方案、上传文档等操作需要 **architect** 或更高角色 — 请联系我方升级
- 每家事务所数据相互隔离，无法看到其他事务所项目
- Settings 中可查看本所成员列表（只读）

## 已知限制（试用阶段）

详见 [known-issues.md](./known-issues.md)。摘要：

- 邀请同事加入同一事务所：需我方后台协助（正式邀请流程在后续版本）
- Vercel 部署若未配置 S3/R2：上传文件重启后可能丢失 — 生产试用必须配置对象存储
- AI 额度：每事务所每日/每月有调用上限，超限会提示
