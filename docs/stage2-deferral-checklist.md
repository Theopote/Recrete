# 阶段二/三/四 启动决策清单

> 路线图第 6 个月要求：回头看任务书里阶段二（真队列）、阶段三（完整测试）、阶段四（正式 RBAC）等，**按真实用户量决定要不要启动**。  
> 大概率 MVP 试用阶段（1–2 家事务所）**还不需要**。

## 决策门槛（建议）

| 指标 | 暂不启动 | 可考虑启动 |
|------|----------|------------|
| 付费/正式事务所数 | &lt; 3 家 | ≥ 3 家且留存意愿明确 |
| 月活用户 | &lt; 10 | ≥ 10 |
| 并发上传/分析 | 单实例够用 | 频繁超时、需水平扩展 |
| 合规/采购要求 | 无 | 客户要求 RBAC 审计、SLA |

**当前 MVP 目标用户量：1–2 家试用** → 默认 **全部暂缓**。

## 阶段二 · 真队列与基础设施

| 项 | 现状 | 何时需要 | 建议 |
|----|------|----------|------|
| BullMQ + Redis worker | 可选，`REDIS_URL` 启用 | 多实例 Vercel、分析任务积压 | 第 2 家试用仍可用内部 API；≥3 家再评估 |
| S3/R2 对象存储 | 已支持，需配置 | Vercel 生产 | **现在就要**（P0 级部署项，非阶段二） |
| 邮件邀请 / SMTP | 密码重置可选 | 自助 onboarding | 阶段二；试用用 `trial:create-org` |
| Pinecone 向量库 | 可选，本地 TF-IDF 回退 | 知识库规模大 | 暂缓 |

## 阶段三 · 完整测试体系

| 项 | 现状 | 何时需要 | 建议 |
|----|------|----------|------|
| `smoke:core` | 5 项核心检查 | 每次改完 | **保持**，够非程序员验收 |
| `smoke:trial-prep` | 部署前检查 | 每次上线 | **保持** |
| E2E Playwright | 无 | 团队扩大、多人改代码 | 暂缓 |
| CI 全量门禁 | 部分 | 开源或多人协作 | 暂缓 |

## 阶段四 · 正式 RBAC 与成员管理

| 项 | 现状 | 何时需要 | 建议 |
|----|------|----------|------|
| 角色权限矩阵 | 基础 7 角色 | 试用够用 | 暂缓大改 |
| 组织成员 CRUD UI | 无 | 客户自助加人 | 阶段四 |
| 审计日志 | 无 | 企业采购 | 阶段四 |
| 方案审批工作流 | 留言评审 only | 正式签批 | 阶段四 |

1. 按 [feedback-triage-workflow.md](./feedback-triage-workflow.md) 修 **P0/P1 真实反馈**

4. 仅当试用暴露 **多实例/队列** 阻塞时，优先 `REDIS_URL` + worker，而非一口气上阶段二全套
